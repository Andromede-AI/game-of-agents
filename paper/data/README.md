# `paper/data/` — data artifacts behind the paper

This directory contains the data underlying every figure and table in
`paper/main.tex`, plus the derivation notes that fed the prose.
Everything here is generated from `.goa_data/runs/*.json` (local JSON
exports of completed tournament runs) by scripts under `scripts/` and
`analysis/`; each section below names the generator.

All figures are regenerable from the CSVs via
`uv run python scripts/make_figures.py`. Consistency against `main.tex`
is asserted by `tests/test_paper_data_consistency.py`.

---

## Primary tables (figure inputs)

### `run_summary.csv` — 39 rows, one per release-corpus run

The top-level per-run aggregate table consumed by all four figures and
by §4 prose. Generated by `scripts/make_figures.py` during its first pass.

| column | type | meaning |
|---|---|---|
| `run_id` | string | Convex run ID, e.g. `run_k171het0ovc8cg` |
| `condition` | string | Human-readable condition label. One of 15 values — see `tests/test_paper_data_consistency.py::KNOWN_CONDITIONS` |
| `duration_h` | float | Wall-clock run duration in hours. 1h, 3h, or 6h in the current corpus |
| `rep` | int | Replication index within the condition (1-indexed) |
| `agents` | int | Number of agents in the run (typically 6 or 8) |
| `n_games` | int | Total finished 4-player NLHE games scored by the tournament engine |
| `gini` | float | Gini coefficient of final display ratings across agents. Used in the Gini comparison figure |
| `n_offers` | int | Count of marketplace offers created during the run. Authoritative — backfilled from `batch_summary.md`'s `/analysis` endpoint totals. Note: the Convex `listMarketplaceOffers` query paginates at `limit=50`, so local JSON exports under `.goa_data/runs/` undercount `n_offers` for any run with > 50 offers. The CSV is the source of truth; `scripts/make_figures.py` reads `n_offers` from the CSV to bypass the cap. |
| `n_purchases` | int | Count of completed marketplace purchases. Not subject to the `listMarketplaceOffers` cap (fetched via a per-offer indexed query). |
| `buy_rate` | float | `n_purchases / n_offers`. Fig. 3 annotates this above each bar |
| `n_chat` | int | Count of chat messages posted |

### `taxonomy_frequencies.csv` — 292 rows, one per (run, agent)

LLM-as-judge classifier output, one row per agent per run. Input to
Fig. 4. Generated by `scripts/run_taxonomy_classifier.py` (resume-safe
batch driver).

| column | type | meaning |
|---|---|---|
| `run_id` | string | |
| `agent_id` | string | `agent-1` … `agent-N`, `claude-1` … `gpt-N` in hetero runs |
| `model` | string | Anthropic/OpenAI API ID (`claude-sonnet-4-6`, `o4-mini`, `claude-opus-4-7`) |
| `competitive_coding` | float in [0,1] | |
| `marketplace_exploitation` | float in [0,1] | |
| `social_influence` | float in [0,1] | |
| `information_exploitation` | float in [0,1] | |
| `collusion` | float in [0,1] | |
| `confidence` | string | `high`/`med`/`low` — classifier self-reported |
| `data_sparsity_flags` | string | `;`-separated; possible values: `no_marketplace`, `no_chat`, `no_reviews`, `short_transcript`. Empty if the run exercised all channels |
| `classifier_model` | string | Fixed at `claude-sonnet-4-5` for the entire corpus |
| `classifier_version` | string | Fixed at `taxonomy-v1` |
| `input_tokens_est` | int | Estimated input tokens for the classifier call (cost accounting) |

### `chip_games.csv` — 46,720 rows, one per (run, game, seat)

Per-hand chip outcomes, the source of truth for the laddering metric
(§4.1 + appendix §H). Generated by `scripts/extract_chip_games.py`.

| column | type | meaning |
|---|---|---|
| `run_id` | string | |
| `condition` | string | Denormalized for convenience |
| `duration` | string | `1h`/`3h`/`6h` |
| `rep` | int | |
| `game_id` | string | Unique Convex game doc ID |
| `agent_id` | string | Owner of the bot seated at this position |
| `model` | string | |
| `bot_id` | string | Bot bundle ID; the agent may have multiple bots over the run |
| `seat` | float | Seat index 0-3 (float because of an earlier export bug; cast as int on read) |
| `starting_chips` | int | Always 200 in the current corpus |
| `ending_chips` | float | Chip stack after the hand finished (or 0 if busted) |
| `delta` | float | `ending_chips - starting_chips`. Sum over a run gives total chip profit |
| `placement` | float | 1 = 1st place, 4 = 4th place (cast as int on read) |
| `table_size` | int | Always 4 in the current corpus |
| `round_count` | int | Number of betting rounds played before the hand ended |

## Secondary artifacts

### `taxonomy_evidence.jsonl` — 292 lines

Full audit trail for the classifier. One JSON object per (run, agent)
with the per-category `rationale` and `evidence_quoted` fields as they
came back from the Anthropic tool-call. Used by
`scripts/taxonomy_audit_sample.py` to construct the hand-label template.
Stripped of the `classifier_scores` numbers before being shown to the
human auditor.

### `taxonomy_audit_20.md` + `taxonomy_audit_scores.json`

Blind-label template + hidden sidecar for the 20-case classifier
validation audit. The md has empty `case_N:` YAML
blocks; the JSON has the matching classifier scores so that
`scripts/taxonomy_audit_compute.py` can compute κ / Spearman ρ /
agree-within-0.25 once labels are filled in.

### `batch_summary.{md,json}`

Per-condition purchase-rate summary with 95 % bootstrap CIs.
Headline numbers (§4.2):
- B1 cooperative: 12.2 [9.8, 15.5] per 3 h, N=4
- Clean additive: 32.0 [31.2, 32.8] per 3 h, N=4 (2.6 × B1)
- No-reviews + additive: 61.5 [55.0, 68.0] per 3 h, N=2 (5.0 × B1)

### `chip_vs_trueskill.md`

Per-condition Spearman/Kendall correlation between total chip profit
and final display rating (backed by `best_elo` in the export), one row per condition. Generated by
`scripts/chip_vs_trueskill_by_condition.py`. Supersedes the earlier
pre-batch reconciliation note used during the ρ ≈ −0.5 investigation.

### `chat_case_studies.md`

Per-condition chat-transcript summaries, used as quote source for §4
prose and the detailed case studies in appendix §K. Organized by
condition label; each section lists 5–10 verbatim messages with
attributed run_id + agent_id.

### `kf2k_findings.md`, `op47_findings.md`

Deep-dive analyses of the two runs that drive most of the qualitative
§4 discussion: `run_kf2kp5qrz0mmsy` (clean-additive headline, 75 %
review rate, explicit ROI math in chat) and the homogeneous Opus 4.7
B1 run (the 0-offer / "dry powder" monoculture attractor).

### `offer_honesty.md`

Rate at which marketplace offer titles contain rank claims that are
consistent with the seller's actual display-rating position at the time
of the offer. B1 = 3 %; adversarial framing = 14 %.

### `stats_audit.md`, `transcript_scan.md`, `win_source.md`

Internal audit notes: precise transcript-scan counts backing §4.2's
9.4 % / 28.3 % / 5.3 % incentive-awareness rates; per-condition
win-source accounting (tournament vs marketplace contributions to final
score).

### `settlement_examples.md`, `inequality_triangulation.md`, `throughput_normalization.md`

Workshop-revision appendix artifacts generated by
`scripts/reviewer_artifacts.py`: the exact settlement formulas and
worked example; matched-horizon inequality across display rating vs base
score vs payout; and throughput-normalized marketplace metrics for the
main settlement / review contrasts.

### `coordination_power.md`, `confounded_additive_diagnostic.md`, `laddering_robustness.md`, `aux_channel_ablation.md`

Additional reviewer-facing robustness notes generated by
`scripts/reviewer_artifacts.py`: coordination nulls and approximate
power / MDE, the 881-purchase confounded-additive diagnostic, a
compact laddering robustness table keyed to the permutation-family
analysis, and the out-of-corpus channel-ablation control status/metrics.

## Regenerating from scratch

```bash
# 1. Primary CSVs (from .goa_data/runs/*.json)
uv run python scripts/make_figures.py               # run_summary.csv + figures
uv run python scripts/extract_chip_games.py         # chip_games.csv
uv run python scripts/run_taxonomy_classifier.py    # taxonomy_frequencies.csv (~$5)

# 2. Per-condition derived tables
uv run python scripts/batch_analysis.py             # batch_summary.{md,json}
uv run python scripts/chip_vs_trueskill_by_condition.py  # chip_vs_trueskill.md

# 3. Laddering metric + null
uv run python scripts/laddering_rate.py --n-null 10000 --seed 17
uv run python scripts/laddering_sensitivity.py

# 4. Validate
uv run python scripts/reviewer_artifacts.py
uv run pytest tests/test_paper_data_consistency.py -v
```
